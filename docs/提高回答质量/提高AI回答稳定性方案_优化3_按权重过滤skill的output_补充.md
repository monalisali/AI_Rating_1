### 1. Jaccard相似系数（用于实体重叠度）

**定义**：Jaccard系数衡量两个集合的相似程度，等于**交集大小除以并集大小**。

**公式**：
\[
J(A,B) = \frac{|A \cap B|}{|A \cup B|}
\]

**取值范围**：0（完全不重叠） ~ 1（完全相同）。

**为什么用于实体重叠度**：用户问题与产物文本在词级别的共现程度，可以反映信息的相关性。如果两者共用的关键词越多，很可能产物的内容与用户问题直接相关。

**计算步骤**（中文环境）：
1. 对文本 `A` 和文本 `B` 分别进行分词（使用jieba）。
2. 过滤停用词（如“的”、“了”、“是”）以及单字符。
3. 将剩下的词转化为集合（自动去重）。
4. 计算交集大小和并集大小，套用公式。

**示例**：
- 用户问题分词集合 Q = `{居民, 企业, 税率, 7.5%}`
- 产物文本分词集合 P = `{居民, 企业, 适用, 15%, 税率}`
- 交集 = `{居民, 企业, 税率}` → 大小 3
- 并集 = `{居民, 企业, 税率, 7.5%, 适用, 15%}` → 大小 6
- Jaccard = 3/6 = 0.5

代码实现（Python）：
```python
def jaccard(set1, set2):
    if not set1 and not set2:
        return 0.0
    inter = len(set1 & set2)
    union = len(set1 | set2)
    return inter / union
```

### 2. TF-IDF（词频-逆文档频率）与局部IDF

#### 2.1 IDF 的定义
**IDF (Inverse Document Frequency，逆文档频率)** 衡量一个词在整个文档集合（语料库）中的稀有程度。稀有的词往往承载更独特的信息，对区分文档更重要。

**公式**：
\[
\text{IDF}(t) = \log \frac{N}{1 + \text{DF}(t)}
\]
- \( N \)：语料库中的文档总数
- \( \text{DF}(t) \)：包含词 \( t \) 的文档数量（文档频率）

**为什么加1**：避免分母为0（当某词从未在语料中出现时，DF=0，此时公式变为 log(N/1)，可接受）。

**特点**：
- 常用词（如“企业”、“的”）出现在几乎所有文档中 → DF ≈ N → IDF ≈ 0
- 罕见词（如“两免三减半”、“西部大开发”）只出现在少数文档中 → DF 小 → IDF 大

#### 2.2 “局部IDF”的含义

常规TF-IDF需要预先扫描全部知识库（全局语料），这在你那里不现实。因此我们使用 **局部IDF**，即仅以**当前用户问题检索到的文章集合**作为语料，计算这些文章中的词频信息。

**优点**：
- 无需任何离线预处理，完全在线。
- 语料规模小（通常几十篇文章），计算快。
- 能自动适应当前问题的上下文：在讨论西部大开发时，“西部大开发”这个词在当前文章集合中出现频率低，IDF高，因而被赋予高权重；而如果用户问的是增值税问题，相同文章集不会包含“西部大开发”，则其IDF无关紧要。

#### 2.3 计算局部IDF的步骤（在代码中体现）

1. **收集文档**：当前用户问题检索到的所有文章（`articles`列表）。
2. **对每篇文章的内容进行分词、去停用词**，并**对每篇文章内的词进行去重**（因为DF统计的是“有多少篇文章包含该词”，同一篇文章内多次出现只计一次）。
3. **统计每个词出现在多少篇文章中** → 得到 DF。
4. **设总文章数为 N**，对每个词计算 `idf = log( N / (1+DF) )`。
5. **存储为字典** `{词: idf值}`。

**示例**（假设当前检索到5篇文章）：

| 词 | 包含该词的文章数(DF) | IDF = log(5/(1+DF)) |
|----|---------------------|---------------------|
| 企业 | 5 | log(5/6) ≈ -0.08 → 取0 (一般不使用负值，可设下限0) |
| 西部 | 1 | log(5/2) ≈ 0.92 |
| 大开发 | 1 | 0.92 |
| 减半 | 2 | log(5/3) ≈ 0.51 |

#### 2.4 计算产物的“平均IDF”

对于给定产物文本（如一条条件-结论对的`condition + conclusion`），我们计算其中所有词的IDF平均值。

**步骤**：
1. 对产物文本分词、去停用词（**不去重**，保留词频，因为出现的次数也隐含信息）。
2. 对每个词，在IDF字典中查找其IDF值（若词不在字典中，IDF=0）。
3. 计算所有词IDF的总和，除以词的总个数，得到平均IDF。
4. 将该平均值作为“逻辑独特度”的原始分数。

**示例**：
产物文本为“居民企业执行西部大开发优惠政策”，分词后得到 `[居民, 企业, 执行, 西部, 大开发, 优惠政策]`。  
假设IDF字典中：
- 居民: 0.10
- 企业: 0.00
- 执行: 0.00
- 西部: 0.92
- 大开发: 0.92
- 优惠政策: 0.20

平均IDF = (0.10+0.00+0.00+0.92+0.92+0.20)/6 = 2.14/6 ≈ 0.357

#### 2.5 为什么要归一化

不同类别产物（如条件-结论对平均IDF的绝对数值可能高于政策场景标签），为了在同一类别内公平排序，我们需要对每个类别各自的`平均IDF`集合进行线性归一化到 `[0,1]`。归一化公式：

\[
\text{norm} = \frac{x - \min}{\max - \min}
\]

这样，每个类别中IDF最高的产物得1，最低得0。
