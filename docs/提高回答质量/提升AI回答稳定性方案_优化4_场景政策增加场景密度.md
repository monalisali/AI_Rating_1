# 开发文档：政策场景自动语义扩展模块

> **面向开发人员与Claude Code**  
> 本文档描述如何对政策场景标签进行自动语义扩展，通过关联高相似度的条件-结论对和概念关系断言，增强场景文本的信息密度，从而提升核心场景在多维评分中的排名，避免被误截断。本方案不依赖硬编码白名单，完全基于向量语义匹配。

---

## 1. 背景与目标

### 1.1 问题
政策场景标签（如“西部大开发优惠”）通常仅含几个汉字，信息量极少。在多维评分（语义相似度、实体重叠、逻辑独特度、结构重要性）中，这类短标签往往得分偏低，导致核心场景在排序截断时被错误丢弃（例如西部大开发优惠排名第20位）。

### 1.2 解决思路
利用四个Skill中已经产出的**条件-结论对**和**概念关系断言**。这些产物包含了与政策场景紧密相关的详细规则文本。通过向量语义匹配，为每个政策场景自动找到最相关的若干条条件-结论对和概念关系断言，将其文本内容拼接到场景标签后面，形成**扩展后的场景表示**。然后用这个扩展表示计算多维评分，使核心场景的得分显著提升。

### 1.3 目标
- 无需任何人工白名单或关键词映射。
- 完全自动化，适应任意用户问题。
- 显著提高核心政策场景的评分，保证其进入Top N保留列表。

## 2. 输入数据

### 2.1 政策场景列表
```python
scenes = [
    "西部大开发优惠",
    "高新技术企业优惠",
    "经济特区高新技术企业过渡性优惠",
    # ... 其他场景
]
```
每个元素为字符串（场景标签）。

### 2.2 条件-结论对列表
```python
pairs = [
    {
        "id": "pair_001",
        "text": "企业既符合西部大开发15%优惠税率条件...可以按照企业适用税率（15%）计算的应纳税额减半征税",
        "vector": [...],   # 已预计算，768/1024维
    },
    # ...
]
```
其中 `text` 字段为 `condition + " → " + conclusion`。

### 2.3 概念关系断言列表
```python
rels = [
    {
        "id": "rel_001",
        "text": "西部大开发15%优惠税率 与 定期减免税减半期 相关但不可等同",
        "vector": [...],
    },
    # ...
]
```
`text` 字段为 `entity_a + " " + entity_b + " " + relation_type + " " + evidence`（可自定义，但应包含关键信息）。

### 2.4 向量编码器
使用 BGE-M3 模型（与检索环节相同），要求能够对文本生成向量。

---

## 3. 核心算法：语义扩展

### 3.1 步骤
1. 为每个政策场景生成向量（若未缓存）。
2. 遍历所有条件-结论对和概念关系断言，计算余弦相似度。
3. 按相似度降序排序，取 Top K（例如 K=3）作为扩展候选。
4. 可选：设定相似度阈值（如 0.6），低于阈值的候选不采用。
5. 将场景标签与选中候选的文本拼接，生成扩展文本。

### 3.2 扩展文本格式
```
{场景标签}。相关规则：{候选1文本}。{候选2文本}。{候选3文本}
```
示例：
```
西部大开发优惠。相关规则：企业既符合西部大开发15%优惠税率条件，又符合各项税收优惠条件，且处于定期减免税的减半期内，可以按照企业适用税率（15%）计算的应纳税额减半征税（即适用7.5%税率）。
```

### 3.3 代码实现（伪代码）

```python
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

def expand_scenes(scenes, pairs, rels, top_k=3, sim_threshold=0.6):
    """
    scenes: list of str
    pairs: list of dict, each with keys 'id', 'text', 'vector'
    rels: list of dict, each with keys 'id', 'text', 'vector'
    top_k: 每个场景取最相关的候选数量
    sim_threshold: 最低相似度阈值，低于此值不扩展
    return: dict {scene: expanded_text}
    """
    # 为场景生成向量（若你已有全局缓存，可复用）
    scene_vectors = {}
    for scene in scenes:
        scene_vectors[scene] = encode(scene)  # 假设 encode 函数调用 BGE-M3

    expanded = {}
    for scene, s_vec in scene_vectors.items():
        candidates = []
        # 评估条件-结论对
        for p in pairs:
            sim = cosine_similarity([s_vec], [p['vector']])[0][0]
            candidates.append((sim, p['text']))
        # 评估概念断言
        for r in rels:
            sim = cosine_similarity([s_vec], [r['vector']])[0][0]
            candidates.append((sim, r['text']))
        # 排序
        candidates.sort(key=lambda x: x[0], reverse=True)
        # 筛选
        selected = [text for sim, text in candidates[:top_k] if sim >= sim_threshold]
        if selected:
            expanded_text = scene + "。相关规则：" + "。".join(selected)
        else:
            expanded_text = scene   # 无有效候选时保留原标签
        expanded[scene] = expanded_text
    return expanded
```

### 3.4 性能优化
- **缓存**：场景向量、条件-结论对向量、概念断言向量可以离线预计算并存储（例如存为JSON或pickle），只要内容不变即可复用。
- **增量更新**：当知识库更新导致条件-结论对或概念断言内容变化时，重新计算向量即可。
- **批处理**：可一次性计算所有场景与所有产物的相似度矩阵（若数据量小）。

## 4. 集成到多维评分流程

在原有的多维评分模块中，将政策场景的表示从原始标签替换为扩展文本。

### 4.1 修改点
```python
# 原评分代码
for scene in scenes:
    score = multi_factor_score(scene, ...)   # 使用原始标签

# 改为
expanded_map = expand_scenes(scenes, pairs, rels)
for scene in scenes:
    enriched = expanded_map[scene]
    score = multi_factor_score(enriched, ...)
```

其余部分（计算语义相似度、实体重叠、逻辑独特度、结构重要性）保持不变，因为所有这些函数都接受一个文本字符串作为输入。

### 4.2 注意事项
- 扩展后的文本可能较长（数百字），但政策场景数量很少（通常<30），对整体性能影响可忽略。
- 逻辑独特度（局部IDF）基于当前文章集，扩展文本中的词语可能包含更多常见词，可能略微降低该维度分数。但语义相似度和实体重叠度的提升通常远大于此损失。

## 6. 配置参数推荐

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| top_k | 3 | 每个场景最多关联3个候选 |

| 向量模型 | BGE-M3 | 与检索系统一致 |

可根据实际测试微调。

---

## 7. 异常处理与降级

- 若场景无任何候选满足相似度阈值（例如新创场景），则保留原始标签，不扩展。
- 若条件-结论对或概念断言列表为空，扩展模块跳过，直接返回原始标签。
- 所有异常情况记录日志，不影响主流程。
