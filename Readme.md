# 1. 文件解释
## 1.1 提示词md文件
1. Prompt.md
 - 最后喂给法规政策助手（5007），适配所有问题的提示词
2. Prompt_模型评测原始.md
- docs/模型打分
- 模型打分的目的是给定一个问题（现行企业所得税法下，居民企业什么情况下取得的所得可能适用7.5%税率？），提示词中给到所有引用文章ntpsid，然后在AI回答问题后进行打分。Prompt_模型评测原始.md 就是模型打分最原始的提示词
3. Prompt_answer_staility.md
- docs/提高问题质量
- “回答稳定性”会通过4个skill产生一份提示词，要把它和“Prompt_模型评测原始.md”结合起来（在其基础上加上了5“推理依据”章节）。然后再用Prompt_answer_staility.md放到“模型评测.xlsx”文件的“提示词”栏位，重新进行评分。这样就能对比修改前后的评分了。

# 2. 功能使用
## 2.1 批量评估
它有2个功能
1. 通过AI获取回答，然后对回答进行打分
2. 自动对提示词进行优化

### 2.1.1 操作流程
1.上传\docs\问题评分_提示词自动优化\ 下"Tax知识库回答及问题反馈"开头的excel文件 
2.只勾选UI上的“启用语义对比打分” —— **只会打分，不自动优化提示词**
3.同时勾选“启用语义对比打分” 和 “启用提示词优化” —— **打完分后，如果触发了条件就会自动更新提示词**，单选“启用提示词优化”是不会触发提示词优化的，因为优化的触发条件要看打了几分

## 2.2 快速评分
快速对1个问题进行打分。UI上输入问题、AI回答、建议回答即可

## 2.3 模型打分
用相同的问题、相同的提示词、选用不同的模型获取AI回答然后进行打分，来查看模型之间的差异。而且为了规避文章召回率的对回答质量的影响，提示词中hardcode了回答所需的ntpsid

### 2.3.1 操作流程
1. 上传 C:\D\AI_Rating_1\docs\模型打分 下的 “模型评测.xlsx”（系统只会读取第一个sheet）
   - 表格中的“提示词”栏位就是系统会使用的提示词
2. “并发线程数”一般选择4
3. 完成后系统会自动在ouputs\模型打分结果\ 下生成结果

## 2.4 回答稳定性
目的：通过4个skill对回答所需文章提取：条件-结论对、政策场景、概念断言、时间约束 4个维度的动态提示词。然后把它拼接到最后的提示词中，再让AI回答问题。

### 2.4.1 操作流程
1. 上传 C:\D\AI_Rating_1\docs\提高回答质量\ 下的“回答稳定性.xlsx”。 **注意：** 提示词栏位中的Prompt只是用来获取文件，然后把文章喂给skills（python代码）来获取动态提示词的。但如果“文章内容”栏位有值，系统就会直接用它而不会再去获取文章了，这样可以节省大量时间。
2. 完成后系统会自动在ouputs\回答稳定性结果\ 下生成结果
